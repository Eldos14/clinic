# Клиника — Личный кабинет пациента и панель управления

Простой PHP/MySQL проект для медицинской клиники с ролью пациента, врача и администратора.

## Что реализовано

- Личный кабинет пациента
- Просмотр и редактирование данных пациента
- Запись к врачу
- Страница списка врачей
- Уведомления пациента
- Страница врача с расписанием и доступом к пациентам
- Админ-панель для управления врачами, пациентами и расписанием
- AI-помощник на главной странице с поддержкой Deepseek / OpenAI и локальным резервным ответом

## Структура проекта

- `index.php` — главная страница пациента
- `header.php` — общий шапка сайта
- `db.php` — подключение к базе данных
- `myDetails.php` — профиль пациента
- `myClinic.php` — информация о клинике пациента
- `doctors.php` — список врачей для пациента
- `doctor.php` — страница записи к врачу
- `patient_history.php` — история визитов пациента
- `notifications.php` — уведомления пациента
- `ai_assistant.php` — серверная логика AI-помощника
- `add_appointment.php` — форма записи на приём
- `add_patient.php`, `add_appointment.php` — вспомогательные формы
- `doctor/` — раздел для врача
- `admin/` — раздел для администратора
- `reg_login/` — регистрация и вход
- `css/` — стили
- `images/` — изображения и иконки

## Настройка и запуск

1. Скопируйте проект в веб-папку типа `htdocs/clinic`.
2. Настройте Apache / Nginx и PHP.
3. Создайте базу данных MySQL и импортируйте нужные таблицы.
4. Откройте `db.php` и установите параметры подключения:
   - `$server`
   - `$database`
   - `$username`
   - `$password`
5. Запустите сайт в браузере.

## Переменные окружения для AI

Для работы AI-помощника можно использовать сначала Deepseek, затем OpenAI как резерв:

- `DEEPSEEK_API_KEY` — ключ API Deepseek
- `DEEPSEEK_API_URL` — (опционально) URL Deepseek API, по умолчанию `https://api.deepseek.ai/v1/complete`
- `OPENAI_API_KEY` — ключ API OpenAI
- `DEEPSEEK_DEBUG=1` или `AI_DEBUG=1` — включить отладку ответа AI

Если ключи не заданы или сервис не отвечает, используется локальный резервный режим.

## Где смотреть логи AI

При работе AI-помощника можно включить отладку, чтобы получить дополнительные данные:

- `deepseek_status` — HTTP-код ответа от Deepseek
- `deepseek_error` — ошибка при выполнении запроса
- `deepseek_response` — текст ответа от Deepseek

## Важно

Этот проект не является медицинским продуктом. AI-помощник даёт лишь ориентировочные подсказки; окончательное решение должно принимать медицинский специалист.
